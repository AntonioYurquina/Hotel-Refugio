TP Integrador — Hotel

Trabajo Práctico Integrador — Desarrollo de un Sitio Web de Hotel

Resumen de requisitos, funcionalidades y tecnologías sugeridas para el proyecto.

Requisitos generales

  • Asistencia: 80% a clases teóricas y prácticas.
  • Entrega individual o grupal (máx. 2 integrantes).
  • El sitio debe permitir uso en localhost y deploy en la nube (GitHub Pages u otro).
  • Aplicar buen diseño UX, validación de todos los campos e inclusión de al menos 2 APIs Web.
  • Base de datos: relacional (SQL) o documental (NoSQL).
  • Documentación básica de tecnologías usadas.

Funcionalidades obligatorias

Usuario

  • Ver habitaciones y servicios.
  • Realizar reserva (form con validación).
  • Consultas por mail (form o integración con API de email).

Operador

  • Consultar habitaciones (mapa/planilla).
  • Consultar y liberar reservas.
  • Abrir / cerrar habitación.
  • Procesar pago y responder mails.

Administrador

  • CRUD de habitaciones.
  • CRUD de operadores.
  • Consultas parametrizadas y gráficos (estadísticas).

Tecnologías y buenas prácticas sugeridas

  • Frontend: HTML5, CSS3 + framework (Bootstrap, Tailwind) y JS moderno (ES6+).
  • Backend: Node.js / Express, o PHP/Laravel, o Python/Flask según lo visto en clase.
  • BD: MySQL/Postgres o MongoDB.
  • APIs: mínimo 2 APIs Web externas (a modo de ejemplo se usan Unsplash y GitHub en esta página).
  • Validaciones: cliente y servidor; control de subida de imágenes y sanitización.
  • UX: navegación clara, formularios con feedback, accesibilidad básica.

Despliegue y documentación

  • Deploy recomendado: GitHub Pages (frontend), Heroku / Vercel / Render para backend o contenedor en la nube.
  • Incluir README con: tecnologías, estructura, comandos para levantar en localhost, endpoints y ejemplos de uso.

Cronograma

Presentaciones desde 18 de octubre hasta 12 de noviembre inclusive. Planificar entregables e hitos.
